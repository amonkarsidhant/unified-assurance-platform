# Contribution Standard (Practical)

This repository favors concise, auditable, and executable changes.

## Scope
Applies to:
- Major design documents in `docs/`
- Release/report artifacts generated for stakeholder review
- Significant policy/process proposals

## Required for major docs/reports
1. Clear objective and target audience.
2. Explicit assumptions and out-of-scope notes.
3. Evidence references (tests, artifacts, policy checks).
4. **Self-Reflection block** using `templates/self-reflection-template.md`.

## Definition of done for doc/report changes
- Content is practical and implementation-oriented.
- Any new process requirement is linked to a script, policy, or measurable check where possible.
- Security/compliance/governance/operability impacts are called out.

## Review expectations
- Prefer small, reviewable increments.
- Avoid ambiguous language; be specific on ownership and acceptance criteria.
